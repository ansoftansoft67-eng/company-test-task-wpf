# 📌 Test Task — WPF + MVVM

## 🎯 Цель

Реализовать desktop-приложение для **управления задачами** на WPF с использованием паттерна MVVM.  

---

## 🧩 Задание

Создать приложение **«Task Manager»** (менеджер задач).

---

### 📋 Модель задачи

| Поле | Тип | Описание |
|------|-----|----------|
| `Id` | Guid | Уникальный идентификатор |
| `Title` | string | Название задачи |
| `Description` | string | Описание (опционально) |
| `Status` | enum | Active / Completed |
| `Priority` | enum | Low / Medium / High |
| `DueDate` | DateTime? | Срок выполнения (опционально) |
| `CreatedAt` | DateTime | Дата создания |

---

### ⚙️ Функционал

| Функция | Описание |
|---------|----------|
| ➕ CRUD задач | Добавление и **редактирование** — в отдельном **модальном окне/диалоге** |
| 🗑️ Удаление | С **подтверждением** (диалог «Вы уверены?») |
| ✅ Смена статуса | Отметка выполненной / возврат в активные |
| 🔍 Фильтр | Все / Активные / Завершённые |
| 🔎 Поиск | По названию и описанию (в реальном времени) |
| 📊 Сортировка | По дате создания, приоритету или сроку |
| ⚠️ Просроченные | Визуальное выделение задач с истёкшим сроком |
| ✔️ Валидация | Title обязателен; DueDate не может быть в прошлом при создании |
| 🔄 Обработка ошибок | При загрузке/сохранении JSON — сообщение пользователю |

---

### 📋 Требования к реализации

| Требование | Описание |
|------------|----------|
| **WPF** | Приложение на Windows Presentation Foundation |
| **MVVM** | Паттерн Model-View-ViewModel, **без кода в code-behind** (кроме инициализации) |
| **ObservableCollection** | Коллекция задач для автоматического обновления UI |
| **INotifyPropertyChanged** | Уведомление об изменении свойств |
| **ICommand** | Команды с **CanExecute** (кнопки неактивны, когда действие невозможно) |
| **IValueConverter** | Минимум 2 конвертера (приоритет→цвет, дата→строка, просрочено→стиль) |
| **Сохранение в JSON** | Загрузка при старте (**async**), сохранение при изменениях |
| **ICollectionView** | Фильтрация и сортировка без пересоздания коллекции |

---

## 🧰 Технологии

| Компонент | Стек |
|-----------|------|
| UI | WPF, XAML |
| Архитектура | MVVM, C# |
| Хранение | JSON-файл (System.Text.Json или Newtonsoft.Json) |

Можно использовать любые дополнительные библиотеки.

## 📦 Что проверяется

- ✅ **MVVM** — строгое разделение, диалоги через ViewModel
- ✅ **Binding** (привязки, конвертеры, триггеры)
- ✅ **Архитектура** — Model / ViewModel / View, сервис для JSON
- ✅ **Коллекции** — ObservableCollection, ICollectionView
- ✅ **Команды** — CanExecute, передача параметров
- ✅ **Валидация и обработка ошибок**
- ✅ **Чистота кода**
